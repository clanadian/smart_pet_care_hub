# Raspberry Pi 4 - 중앙 서버

Smart Pet Care Hub의 중앙 허브(Hub) 역할을 담당했던 Raspberry Pi 4 서버입니다.

## ⚠️ 코드 유실 안내

해당 코드는 라즈베리파이 SD카드에서 직접 개발되었으며, 별도의 버전 관리나 백업이 이루어지지 않은 상태로 작업되었습니다. 프로젝트 종료 후 SD카드 데이터가 소실되면서 서버 코드 원본을 복구할 수 없게 되어, 이 저장소에는 실제 코드가 포함되어 있지 않습니다. 

## 1. 담당 역할

- **중앙 허브**: Arduino(Wi-Fi)와 STM32(Bluetooth) 양쪽으로부터 데이터·명령을 중계
- **데이터 수집 및 저장**: 센서 데이터를 MariaDB(내부 DB)의 `sensor_log` 테이블 등에 기록
- **웹 서버**: 웹 대시보드(PC/태블릿)에 HTTP로 실시간 상태·그래프·이력 제공
- **웹캠 스트리밍**: USB 웹캠 영상을 대시보드에서 실시간으로 확인 가능하도록 중계
- **원격 제어 중계**: 웹/안드로이드 앱에서 들어온 급식·LED·부저 등 제어 명령을 Arduino/STM32로 전달

## 2. 시스템 동작 흐름

1. 센서 데이터 수집 (Arduino → 서버)
2. 서버로 전송 (`[XXX_AND]SENSOR@...` 프로토콜 파싱)
3. 임계값 판단 (예: 수위 < 30% 또는 온도 > 35°C → 위험 상태)
4. 경보 발생 및 DB 저장 (RGB LED 제어 명령 회신, MariaDB 기록)
5. 원격 제어 명령 수신 시 해당 액션 수행 (급식, LED, 부저, 비상 정지 등)
6. 웹 대시보드 갱신 (그래프, 테이블, 카메라 화면 갱신)

## 3. 연결 구조

| 대상 | 통신 방식 | 용도 |
|---|---|---|
| Arduino UNO | Wi-Fi | 센서 데이터 수신, 급식/LED/부저 명령 송신 |
| STM32 F411RE | Bluetooth | 상태 데이터 전달, 비상 알림(SOS) 수신 |
| USB 웹캠 | USB | 실시간 영상 스트리밍 |
| 웹 대시보드 | Wi-Fi (HTTP) | 실시간 상태 조회 |
| 안드로이드 앱 | Bluetooth | 원격 제어, 상태 확인 |

## 4. 사용 스택

- Raspberry Pi 4, MariaDB (내부 데이터베이스)
- PHP 기반 스크립트로 센서 데이터 저장 처리 (`save_sensor.php` 등, 발표 로그 기준)
- 웹 대시보드는 HTML 기반 페이지에서 실시간 그래프·테이블·영상 제공
